Ruby Classes





This week's techincal topic pertains to a critical concept in Ruby programming, which is the utilization of classes

So, what exactly are classes? Very simply, think of a class as a group of like items/things. Consider the animal kingdom, in which science separates groups of animals into classes based on their characteristics. These classes include birds, reptiles, mammals, and so on, with each class containing species that share similar characteristics. For example, laying eggs is a characteristic which all birds share, and is therefore one reason why birds are grouped together in the same class.

This concept is utilized in programming for much the same reason. By creating classes in our programs, we enable the creation of objects which are instances of these classes. These objects share many similar characteristics, and also behave alike in that they can perform the same methods.

So that our program can remember these various properties about Bob and his friends, we'll need to store their values inside variables. Remember, a variable is just a placeholder for a value, which can be accessed/manipulated during a program. The extent of that access/manipulation is contingent upon another importance concept in programming: variable scope.

Why is variable scope important? Sometimes we may not want all of our variables to be accessible by every part of the program at a given time. Placing restrictions on this access keeps our program structured, and also allows for secure expandability later on. There are four main types of variables that we use in the creation of our class:

  • Global: A global variable is one that is the most accessible, including across all methods and all classes within a program. They are preceded by a $.

  • Class: A class variable is available across different objects within a class, but not across different classes. They are preceded by @@.

  • Instance: Instance variables can be utilized across different methods within a class. They are preceded by @.

  • Local: And finally, a local variable is one that can only be accessed inside the method in which they were defined. Local variables are preceded with nothing.
