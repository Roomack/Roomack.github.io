Week 2: CSS Positioning



As our cohort fine tunes our skills in positioning elements with CSS, I'd like to take some time in explaining the distinction between the four potential values for the "position" property:

Static

Every element that is added to a webpage has a default position value of static when it is created. This essentially means that the element has no specific positioning instructions. In the example below, you'll see that there is no position property listed for either Div1 or Div2 (which demonstrates that this is the default value). Items with a static position will be placed in the uppermost left corner of the page. As items are added to the page, they will be placed on their own row, going vertically down the page.




------------------------

Fixed

An element's position property is set to "fixed" when the developer intends on having the element maintain the same position within the browser window (even if the user is scrolling). This is often used in the case of headers and footers on a webpage, such as the footer for this page. When using the "fixed" position, the developer must also specify where on the page they would like for the element to be placed. In the case of the footer for this site, the "bottom" property is set to "0px", meaning the footer should be placed at the very bottom of the page (see below).




------------------------

Relative

Elements are positioned using "relative" when the developer wants to place the item relative to itself (or relative to where it would have normally fallen). When setting the position to "relative", the top, bottom, left, and/or right attributes must also be set in order to give the item placement. If not, the element will be placed as if it were static. Something to also remember is that any element that is postioned in this fashion will display ON TOP of any other statically positioned element. ----------------

Absolute

Using the "absolute" position, a developer has the power to place an element in a very specific place on the page. When setting the position to "absolute", the element is removed from the normal "flow" of the page, meaning that it isn't affected by, nor does it affect other elements. Again, the top, bottom, left, or right attributes must be defined to give the item position, and these values are relative to the nearest item on the page with "relative" or "absolute" positioning.
